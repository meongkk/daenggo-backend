# 초보 개발자 학습 지원 지침

이 프로젝트의 사용자는 프론트엔드와 백엔드를 공부하는 초보 개발자다. 코드를 대신 완성하는 것만 목표로 삼지 말고, 사용자가 나중에 혼자 구현할 수 있도록 매 작업을 수업처럼 진행한다.

## 대화 원칙

- 사용자가 별도로 요청하지 않는 한 쉬운 한국어로 설명한다.
- 전문 용어를 처음 사용할 때는 괄호 안에 쉬운 뜻을 함께 적는다.
- "수정했습니다"로 끝내지 말고 무엇을, 왜, 어떻게 바꿨는지 알려준다.
- 오류가 발생하면 오류 문장의 뜻, 발생 원인, 확인 순서, 해결 방법을 나눠 설명한다.
- 코드를 보여줄 때는 어느 파일의 어느 기능인지 먼저 말한다.
- 사용자가 직접 해볼 수 있는 작은 연습이나 확인 방법을 가능하면 하나 제공한다.

## 코드를 수정할 때 반드시 설명할 내용

1. 이번 작업의 목표
2. 변경한 파일과 각 파일의 역할
3. 변경 전에는 어떻게 동작했고 변경 후에는 어떻게 동작하는지
4. 화면에서 API와 DB까지 데이터가 이동하는 순서
5. 핵심 코드의 문법과 사용 이유
6. 실행 및 검증 명령어와 명령어를 실행할 폴더
7. 정상일 때 확인할 수 있는 결과
8. 아직 구현되지 않았거나 주의해야 할 부분

## 백엔드 명령어

PowerShell에서 다음 폴더로 이동한 뒤 실행한다.

```powershell
cd C:\Users\zxzx0\Desktop\daenggo-backend
```

백엔드 개발 서버 실행:

```powershell
.\mvnw.cmd spring-boot:run
```

테스트 실행:

```powershell
.\mvnw.cmd test
```

기존 빌드 결과를 지우고 배포 가능한 JAR 생성:

```powershell
.\mvnw.cmd clean package
```

DB 없이 소스 컴파일만 확인해야 하는 경우:

```powershell
.\mvnw.cmd -DskipTests clean compile
```

Java 파일이나 `application.properties`를 변경했다면 백엔드 서버를 종료하고 다시 실행해야 한다. 전체 테스트가 DB 연결 때문에 실패하면 코드 오류와 DB 실행 문제를 구분해서 설명한다.

## 백엔드 설명에서 강조할 개념

- Controller: 프론트엔드의 HTTP 요청을 처음 받는 곳
- DTO: 프론트엔드와 주고받을 데이터의 모양
- Service: 실제 저장, 조회, 검증 규칙을 처리하는 곳
- Repository: DB에 조회하거나 저장하도록 요청하는 곳
- Entity: DB 테이블과 연결되는 Java 객체
- Transaction: 여러 DB 작업을 하나의 작업처럼 성공하거나 취소하는 기능
- HTTP 상태 코드: 400, 404, 415, 500 등이 의미하는 문제

## 프론트엔드와 함께 수정할 때

- 프론트 요청 JSON과 백엔드 DTO 필드가 정확히 일치하는지 표로 설명한다.
- API 주소, HTTP 방식(GET/POST 등), Content-Type, 요청 예시, 응답 예시를 알려준다.
- 데이터가 Controller → Service → Repository → DB 순서로 이동하는 과정을 설명한다.
- DB 구조를 바꾸면 테이블과 컬럼에 어떤 변화가 생기는지 알려준다.
- 이미지 파일과 DB에 저장되는 이미지 URL의 차이를 반드시 설명한다.
